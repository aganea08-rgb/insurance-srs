## Спецификация процесса оформления страхового полиса

| ID | Шаг на диаграмме | Описание |
|----|------------------|----------|
| 1 | Подать заявку | Клиент отправляет POST-запрос `/application/statement` на создание заявки |
| 2 | Запросить список документов | Сервер возвращает GET-запросом `/application/form/statement` форму для загрузки документов |
| 3 | Отправка документов | Клиент загружает документы |
| 4 | Документы подгружены? (нет) | Проверка загрузки - документы не соответствуют формату |
| 5 | 400 Bad Request error | Сервер возвращает ошибку 400 о неверном формате документов |
| 6 | Отправить документы (повтор) | Клиент повторно отправляет документы через POST `/documents/upload` |
| 7 | Документы не полные? (да) | Цикл повторной отправки пока документы не будут полными |
| 8 | Документы подгружены? (да) | Документы успешно загружены и прошли проверку формата |
| 9 | Проверка документов | Сервер начинает проверку загруженных документов |
| 10 | Отправить документы на проверку в ГИБДД | Параллельный запрос в ГИБДД для верификации документов |
| 11 | Отправить запрос в АИС на расчет КБМ | Параллельный запрос в АИС для расчета коэффициента бонус-малус |
| 12 | Ответ получен? (нет) | Ответ от внешних систем не получен |
| 13 | Повторный запрос до 3х раз | Сервер повторяет запросы до 3 раз |
| 14 | Сервер временно не доступен | Таймаут - внешние системы не отвечают |
| 15 | 504 Gateway Time Out error | Ошибка 504 при недоступности внешних систем |
| 16 | Ответ получен? (да) | Ответы от внешних систем успешно получены |
| 17 | Расчет стоимости полиса | Сервер рассчитывает стоимость полиса на основе полученных данных |
| 18 | Отображает стоимость полиса | Сервер показывает клиенту рассчитанную стоимость |
| 19 | Стоимость устраивает? (нет) | Клиент не согласен с предложенной стоимостью |
| 20 | Отказ от заключения договора | Клиент отказывается от оформления полиса |
| 21 | Стоимость устраивает? (да) | Клиент согласен со стоимостью полиса |
| 22 | Подписание договора | Клиент подписывает договор |
| 23 | Оплатить договор | Клиент инициирует оплату через POST `/payments/initiate` с телом `{"contractId": "123", "amount": 5000}` |
| 24 | Показывает форму оплаты | Сервер возвращает форму для ввода платежных данных через GET `/payments/form/{paymentId}` |
| 25 | Вводит данные для оплаты | Клиент отправляет платежные данные через POST `/payments/initiate` с телом `{"cardNumber": "****", "expiry": "12/30", "cvv": "***"}` |
| 26 | Данные карты отвечают требованиям? (нет) | Проверка формата платежных данных - данные неверны |
| 27 | 400 Bad Request неверные данные | Ошибка валидации платежных данных (код 400) |
| 28 | Повторный ввод данных карты | Клиент заново вводит данные карты |
| 29 | Данные карты верные? (нет) | Цикл повторного ввода пока данные не станут верными |
| 30 | Данные карты отвечают требованиям? (да) | Платежные данные прошли валидацию |
| 31 | На карте есть деньги? (нет) | Проверка баланса - недостаточно средств |
| 32 | 402 Payment Required error | Ошибка о недостаточном балансе (код 402) |
| 33 | Ввести данные другой карты | Клиент вводит данные другой карты |
| 34 | На карте есть деньги? (да) | На карте достаточно средств |
| 35 | Списание средств | Сервер списывает средства через платежный шлюз |
| 36 | Повторная попытка оплаты | Сервер инициирует повторную попытку оплаты |
| 37 | Договор оплачен | Подтверждение успешной оплаты |
| 38 | Договор заключен | Статус договора меняется на "заключен" |
| 39 | Отправка данных в АИС | Параллельный процесс - сервер отправляет данные о заключенном договоре в АИС |
| 40 | Формирование полиса в pdf | Параллельный процесс - сервер генерирует PDF-файл страхового полиса |
| 41 | Отправка полиса на email | Сервер отправляет готовый полис на email клиента |
| 42 | Получен полис на email | Клиент получает страховой полис по электронной почте |
